# Seller Camera 实验版本归档分支整理 + 本地主线回退准备（R25，非功能包）

## 1) 当前版本正式定位

当前本地版本应明确定位为：**实验版本 / 候选模型归档版本**。  
它不适合作为正式继续开发主线，核心原因是：

1. 包含大量临时实验入口与候选模型路由（用于 admission/runtime 验证），不等于正式产品承诺；
2. 文档链以“可行化/复核/暂停-继续判断”为主，目标是研究闭环，不是产品收敛；
3. 存在多条已暂停/已停止路径，若继续堆在主线，会持续造成“实验线=正式线”的误读风险。

因此当前应先做**实验线归档冻结**，再回到“第一版正式基线”推进后续正式开发。

## 3) 归档边界（保留什么 / 不应误读什么）

### 3.1 应保留的实验资产

1. 多模型临时实验入口与会话路由（仅开发测试用）；
2. BiRefNet tiny / RMBG-2 / Vision 变体接入与 runtime 合同修复路径；
3. `WhiteBackgroundBaseline` 下 R01-R25 文档、JSON、runs/check 证据链；
4. 实验 metadata 字段与日志追踪语义（用于回看来源）。

### 3.2 不应误读成正式主线承诺的内容

1. 临时模型切换入口（不是正式设置系统）；
2. admission 体系内暂停/停止结论（不是产品功能结论）；
3. 仅实验可运行但未产品化收口的候选路径；
4. 任何“可替代主链”的推断性描述。

5. “第一版正式基线”引用点已明确（建议固定为远端分支/标签/commit hash，避免口头版本名）。

**第一版正式基线（仅保留正式拍摄与稳定输出主链）**

1. 明确约定：实验只进 `archive/*` 或 `exp/*` 分支，不直接进正式主线；
2. 正式主线 PR 描述中增加“是否引入实验资产”检查项；
3. 需要回摘实验成果时，按“单点 cherry-pick + 明确产品理由”执行。
