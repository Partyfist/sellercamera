# Seller Camera 新候选准入准备（v1）

## 1) 为什么 attention 暂停后仍要保留 admission 体系

### 1.1 attention 暂停的关键教训
#### 有效经验（必须保留）
1. `admission_v1` 三层样本（hard_gate / focus_comparison / stability_witness）能稳定约束评估范围。  
2. “硬门槛优先 + 稳定样本防回退”可避免只看少数难例的假阳性结论。  
3. 结构化人工复核（r03-r06）能把“有变化”与“可复核改善”分开，减少误判升级。  

#### 低收益经验（下一候选避免重复）
1. 在同一 hard_gate 阻塞位连续多轮补证据但无档位跃迁，边际收益快速下降。  
2. 仅凭 mask 几何变化（如 coverage_ratio 变化）不足以支持升级，必须回到主问题证据。  
3. EV006 单点成功不能外推为 hard_gate 整体可升级。  

#### 纪律层收获（必须继承）
1. “不达门槛不升级范围”的纪律有效避免了高风险扩散。  
2. “可回退、可对照、可停止”确保候选试验失败不会污染主链。  
3. admission 结论与样本边界绑定，防止过度外推。  

### 1.2 为什么现在进入“新候选准备”而不是直接接入
1. attention 已完成终局收口，继续在同一候选上循环不再产生有效增量。  
2. 但 hard_gate 阻塞问题仍客观存在，说明“候选评估需求”仍在。  
3. 因此当前正确动作是：保留 admission 纪律，切换到“下一候选入场标准化准备”。

## 2) 新候选入场标准（必须同时满足）

## 2.1 值得入场的候选
候选需同时满足：
1. **问题命中价值**：对当前 hard_gate 阻塞位（同色分离、复杂背景纹理残留、跨域边界稳定性、多主体聚焦）有明确理论命中理由。  
2. **最小接入可行**：可在 `SegmentationProvider` 边界内完成最小替换，不要求下游联动大改。  
3. **移动端实验可行**：在当前 iOS 设备条件下可稳定推理，不引入不可控运行不稳定。  
4. **评估成本可控**：可在 admission_v1 内做首轮验证，不需要先扩大样本或重建平台。  
5. **失败可回退**：候选实验失败时可立即回退到 Vision 稳定路径。  

## 2.2 当前不值得入场的候选
以下任一命中即不入场：
1. 只能在离线/高算力场景运行，移动端最小实验不可行。  
2. 必须同步重写 refinement / 去污染 / 合成才“看起来有效”。  
3. 无法在 `SegmentationProvider` 内隔离，接入即污染主链。  
4. 没有针对 hard_gate 阻塞位的清晰命中依据，仅“看起来先进”。  
5. 需要重置 admission 门槛或重写红线才能通过评估。  

---

## 3) 下一候选选择优先级框架

按以下顺序筛选（从高到低）：
1. **硬门槛命中能力**：是否最有希望触达当前终局阻塞样本主问题。  
2. **最小接入风险**：是否能在 provider 边界内低风险接入并保持回退。  
3. **稳定样本安全性预期**：是否更可能避免稳定区回退。  
4. **实验成本**：是否能在 admission_v1 内快速给出“继续/停止”信号。  
5. **后续扩展价值**：若通过首轮，是否具备进入更大范围受控实验的潜力。  

建议使用“先过 1-3，再看 4-5”的筛选纪律；未过 1-3 不进入准入实验。

## 5) 下一候选最小实验边界（下一包必须遵守）

## 5.1 允许做
1. 仅在 `SegmentationProvider` 边界内接入一个新候选。  
2. 保留 Vision 默认路径与显式实验开关。  
3. 仅使用 `admission_v1` 做首轮最小准入实验。  
4. 产出最小资产：JSONL 记录、check、人工判读、首轮准入结论。  

## 5.2 不允许做
1. 不重写白底主流程。  
2. 不联动大改 refinement / 去污染 / 合成。  
3. 不同步接第二候选。  
4. 不扩新样本机制、不放宽 admission 门槛。
