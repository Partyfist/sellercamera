# Seller Camera 正式主线收口说明（Vision）

## 1. 当前口径

- 当前正式主线仅保留端侧 Vision 路径。
- 当前正式主线不包含多模型实验入口，也不承诺 Tiny / RMBG-2 / 其他候选模型的端侧可切换能力。

## 2. 为什么收口到 Vision

- 当前阶段目标是保持正式主线稳定、清晰、可维护。
- 历史多模型能力已完成阶段性实验价值，但会增加主线语义混淆和维护负担。
- 产品路线已明确：端侧主线先收口 Vision；高质量白底图后续通过云端完整 BiRefNet 路线推进。

## 3. 实验能力当前状态

- 历史实验能力已归档到独立分支：`archive/experiment-multimodel-r25-20260424`。
- 归档分支用于回看、复盘、择点回摘，不作为当前正式发布线。

## 4. 正式主线开发边界

- 可以继续推进：拍摄体验、参数控制、Vision 相关稳定性与质量改进。
- 当前不应推进：端侧多模型切换、admission/runtime/terminal 新实验包、实验模型路径扩展。

## 5. 协作约定

- 若未来需要恢复某条实验路线，必须通过归档分支显式回摘，并形成单独任务包说明。
- 不在正式主线中以“隐藏入口”方式保留实验功能。
