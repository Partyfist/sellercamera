# Hurricane Camera UI / UX Research

调研时间：2026-05-23  
调研方式：Mac 上的 iPhone 镜像公开界面观察  
调研对象：飓风相机拍摄页公开 UI / UX  
调研边界：仅观察和操作公开界面，不反编译、不抓包、不破解、不读取隐私数据。

## 1. 执行状态

- iPhone 镜像：已打开。
- 飓风相机：已打开并进入拍摄页。
- 截图/录屏落地：未能保存为本地图片或视频文件。
- 本地保存目录：`Reference/HurricaneCamera/`
- 保存失败原因：当前环境下 `screencapture` 返回 `could not create image from display`，Computer Use 可观察镜像截图流，但未提供直接持久化截图到文件的接口。
- 真机相机画面验证限制：通过 iPhone 镜像点击取景/镜头相关区域时，系统弹出“无法从 Mac 使用 iPhone 相机/麦克风”，因此本轮只能观察 UI 结构、控件状态与公开交互，不夸大为真实相机效果验证。

## 2. 拍摄页结构拆解

### 顶部状态区

- 左上保留时间与音频电平样式的 `L / R` 指示，视频/拍摄工具属性明显。
- 顶部中间是模式切换，观察到 `视频 / 照片` 双模式入口。
- 右上集中放置闪光、辅助入口与更多菜单，图标非常克制。
- 顶部整体高度很低，不抢取景画面。

### 取景区

- 取景区域使用深色背景和 3x3 网格。
- 左上有小型直方图/曝光参考浮层，面积很小，不遮挡主体。
- 取景区下部有镜头焦段入口，观察到 `13 / 24 / 48 / 77` 这类焦段胶囊。
- 当前焦段使用绿色强调色，非当前焦段白/灰显示。

### 底部操作区

- 主拍摄按钮居中，始终是视觉主操作。
- 左侧为小预览/素材入口，右侧为最新图片缩略图。
- 参数入口位于拍摄按钮上方一排，保持近手但不盖住拍摄按钮。
- 底部参数入口以图标 + 短中文标签呈现，观察到 `物距 / 对焦 / 白平衡 / 感光 / 快门速度 / 曝光补偿 / 设置` 等入口。

## 3. 参数控制拆解

### WB / 白平衡

- 点击 `白平衡` 后，底部出现横向刻度尺。
- 当前值以浮动数字牌显示，例如 `8903K`。
- 刻度尺位于拍摄按钮下方/周边的低位区域，当前值有一条竖向指针。
- 横向滑动后刻度展开得更明确，出现主刻度标签，例如 `7500K / 8000K / 8500K / 9000K / 9500K / 10000K`。
- 右侧有 `自动` 胶囊开关，开启时使用蓝/绿色高亮，关闭时灰色。
- WB 的专业感主要来自“刻度尺 + 当前值浮窗 + 右侧 Auto 状态”的组合。

### ISO / 感光

- 点击 `感光` 后，同样出现横向刻度尺。
- 当前值以浮动数字牌显示，观察到低 ISO 数值如 `21`。
- `自动` 开关位于右侧，开启时高亮。
- ISO 入口本身在底部参数列中显示当前语义，手动/自动状态通过右侧开关和入口高亮共同表达。

### Shutter / 快门速度

- 点击 `快门速度` 后出现横向刻度尺。
- 当前值以浮窗显示，例如 `1/30`。
- 右侧 `自动` 开关高亮时，表达自动快门/自动曝光相关状态。
- 快门数值没有用大面板，而是保持低位短刻度，主拍摄按钮仍可见。

### EV / 曝光补偿

- 点击 `曝光补偿` 后，不是普通横向尺，而是出现围绕拍摄按钮右侧的大弧形/圆盘式刻度。
- 当前值浮窗观察到 `+1.1`。
- 刻度沿弧形排布，当前值附近有指针线和数字牌。
- 该设计很有“相机拨盘”感，但占用右下角空间明显，视觉存在强风格属性。

### TINT / 色偏

- 本轮公开拍摄页底部未观察到独立 `TINT` 或色偏入口。
- 可能隐藏在更深层设置或不提供独立入口；本轮不做推断。

## 4. AUTO / RESET / LOCK 状态观察

- `AUTO` 主要以右侧胶囊开关表达，而不是每个参数下方的小按钮。
- 开启态有明显高亮填充，关闭态灰色，状态识别强。
- `RESET` 未在本轮已观察参数里作为明显主按钮出现。
- `LOCK` 未在本轮公开操作中观察到明确状态。
- 对 Seller Camera 的启发是：AUTO/RESET/LOCK 不应只是同样的文字，必须在形态、颜色、可点性上区分。

## 5. 动效与手感观察

可观察到的交互规律：

- 点击参数入口后，底部参数控制区在原位替换/展开，不跳转页面。
- 当前 active 参数图标出现冷色光晕或高亮，位置不变。
- 横向刻度尺很低，展开后仍不抢取景区。
- 当前值浮窗跟随中心指针附近，用户无需在刻度堆里找当前值。
- 参数切换不需要进入二级页面，仍保持拍摄页上下文。

无法确认的部分：

- 真实触觉反馈。
- 真实相机画面亮暗/冷暖/快门变化。
- 滑动惯性和吸附的完整体感。
- 录屏级动画细节。

## 6. Seller Camera 可借鉴点

1. **当前值浮窗**
   - Seller Camera 当前五列短竖轮可以继续保留，但应加入更强的当前值锚点，让用户一眼知道正在写入的目标值。

2. **AUTO 状态要像状态，不只是文字**
   - 飓风相机的右侧胶囊开关很清楚。
   - Seller Camera 的 `AUTO / RESET / LOCK` 应继续做形态分层：AUTO 开启态、AUTO 可恢复态、RESET 动作态、LOCK 禁用态必须一眼不同。

3. **参数展开不离开拍摄页**
   - 飓风相机始终把调参压在底部，不进入设置页。
   - Seller Camera 应继续坚持拍摄页内短交互，不做大面板。

4. **焦段入口低调但清楚**
   - `13 / 24 / 48 / 77` 小胶囊足够专业。
   - Seller Camera 后续镜头入口可参考“焦段数字 + active 强调”，不要做大按钮。

5. **小型直方图/曝光参考可后续考虑**
   - 飓风相机把曝光参考压得很小。
   - Seller Camera 如果未来做过曝/偏色提醒，应保持非阻塞和小型化。

6. **active 高亮带轻光晕**
   - 飓风相机 active 参数不靠大卡片，而靠图标高亮和微光。
   - Seller Camera 可以增强 active 列的精致感，但不要变成彩色控制台。

## 7. Seller Camera 不应照搬点

1. **不要直接照搬大弧形 EV 盘**
   - 飓风相机的 EV 圆盘很有风格，但占右下角空间明显。
   - Seller Camera 是商品拍摄工具，五参数同时可见更重要，不宜让单参数占用大面积。

2. **不要把所有参数都改成横向刻度尺**
   - Seller Camera 已经形成五列短竖向控制台，优势是五个参数同时可比较。
   - 横向尺适合单参数沉浸调节，但会降低多参数并列观察效率。

3. **不要复制品牌视觉元素**
   - 不复制图标、动效、配色或品牌符号。
   - 只借鉴结构原则：低位控制、当前值浮窗、状态清楚、取景优先。

4. **不要引入视频工具的复杂度**
   - 飓风相机有明显视频/专业拍摄工具基因。
   - Seller Camera 当前更应服务商品图采集，不需要音频电平、复杂录制状态和影视化信息密度。

## 8. Seller Camera 下一步可落地建议

### 优先级 1：当前值浮窗 / 锚点增强

在五列短竖轮展开时，为 active 参数增加一个更明确的当前值锚点：

- active 列中心值更大、更稳。
- 可在 active 列上方增加小型浮动值牌。
- 数值变化用 monospaced digit 和轻微 opacity/scale 动效。

### 优先级 2：AUTO / RESET / LOCK 形态统一

继续打磨三类控件：

- `AUTO on`：填充高亮，表示已开启。
- `AUTO off`：描边/弱填充，表示可恢复。
- `RESET`：动作按钮，不表现为开关。
- `LOCK`：灰色禁用，不触发写入。

### 优先级 3：参数入口 active 光晕精修

保持 Seller Camera 当前五参数结构，但增强 active 列：

- 轻微边框。
- 暗色玻璃背景。
- 琥珀/绿色少量强调。
- 不用高饱和大面积色块。

### 优先级 4：镜头入口后续单独设计

可参考焦段数字胶囊，但应结合商品拍摄：

- `0.5x / 1x / 2x / 3x` 或等效焦段。
- active 清楚。
- 不遮挡商品主体。

### 优先级 5：后续辅助信息保持小型化

如果接入曝光参考、水平仪、网格、偏色提醒：

- 放小。
- 非阻塞。
- 可忽略。
- 不抢五参数控制台。

## 9. 风险与限制

- 本轮未保存本地截图/录屏文件，原因是当前环境无法从显示创建截图文件。
- 通过 iPhone 镜像无法使用 iPhone 相机/麦克风，真实画面变化、触觉反馈、滑动惯性无法完整验证。
- 对 TINT / RESET / LOCK 的观察不完整，未在公开拍摄页直接看到独立色偏或锁定控件。
- 本报告只作为 UI/UX 借鉴，不构成对飓风相机内部实现的判断。

## 10. 结论

飓风相机最值得 Seller Camera 借鉴的不是具体图标或大圆盘，而是三件事：

1. 参数始终贴近拍摄页，不离开取景上下文。
2. 当前值浮窗非常明确，用户知道正在调什么。
3. AUTO 状态表达强，不只是文字。

Seller Camera 下一步更适合在现有 `EV / WB / TINT / ISO / S` 五列短竖轮基础上，做“当前值锚点 + 状态控件形态统一 + active 高亮精修”，而不是把交互整体改成飓风相机的横向刻度或圆盘。
